Tabs

Tabs organize and allow navigation between groups of content that are related and at the same level of hierarchy.

Usage Note:

  • Use Cases: Course pages (Overview, Syllabus, Resources), User Profiles (Progress, Saved Items), and Admin panels.
  • Best Practices: Keep labels short (3-6 tabs max). Ensure a clear active state and support keyboard navigation (arrow keys).
  • Sub-tabs: For secondary filtering within a tab, use smaller, pill-shaped tabs.

Accessibility (WCAG AA)

  • Color Contrast: All text and icons maintain AA contrast minimums over surfaces in both light and dark themes.
  • Keyboard Navigation: All interactive elements are reachable and operable via keyboard. The tab order follows the visual order, and visible focus rings are present.
  • Screen Readers: Semantic HTML is used. `aria-labels` are provided for icon-only buttons, and descriptive `alt` text for images.
  • Forms: All form inputs are associated with a `label`. Error messages are programmatically linked to their respective inputs.
  • Reduced Motion: Animations and transitions are minimized or disabled if the user has a `prefers-reduced-motion` preference.
